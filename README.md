# API أخبار عربي (يتحدث كل ساعة تلقائيًا)

سكربت بايثون يجلب آخر المقالات من عدة مصادر إخبارية عبر RSS، ويحفظها في ملف
`news.json` واحد. عن طريق GitHub Actions، يعمل هذا السكربت تلقائيًا كل ساعة
ويرفع التحديث، فيصبح رابط الملف الخام على GitHub بمثابة API مجاني وثابت.

## المصادر المضبوطة حاليًا

| المصدر | الحالة |
|---|---|
| الجزيرة | رابط RSS مباشر ومؤكد |
| العربية | رابط RSS مباشر ومؤكد |
| BBC عربي | رابط RSS مباشر (نمط معروف ومستقر لدى BBC) |
| CNBC عربية | لم أجد رابط RSS مباشر مؤكدًا، السكربت يحاول يكتشفه تلقائيًا من الصفحة الرئيسية |
| الحرة | نفس الحالة، اكتشاف تلقائي |
| 964 | نفس الحالة، اكتشاف تلقائي |

الاكتشاف التلقائي يعمل غالبًا (يبحث عن وسم `<link type="rss">` بصفحة الموقع،
وإن لم يجده يجرّب مسارات شائعة مثل `/feed/`). لو فشل مع أحد المصادر الثلاثة،
السكربت لا يتوقف — يكمل باقي المصادر ويسجّل أن هذا المصدر "failed" في
تقرير `sources` داخل ملف الإخراج. لو حصل ذلك، دور على رابط RSS الحقيقي من
موقع المصدر (غالبًا مكتوب بأسفل الصفحة) وأرسله لي لأحدّث الرابط في السكربت.

## خطوات التشغيل (مرة واحدة فقط)

1. أنشئ مستودع (repository) جديد **عام (Public)** على GitHub.
2. ارفع كل ملفات هذا المجلد بما فيها مجلد `.github/workflows` كما هو (لا
   تغيّر اسمه أو مكانه، GitHub يتعرف على الـ workflow من هذا المسار بالذات).
3. روح لتبويب **Actions** في المستودع، وفعّل الـ workflows لو طلب منك ذلك.
4. من تبويب Actions، افتح "Update Arabic News" واضغط **Run workflow** لتشغيله
   فورًا أول مرة بدل انتظار الساعة القادمة.
5. بعد انتهاء التشغيل (دقيقة أو دقيقتين)، سيظهر ملف `news.json` في المستودع.

## رابط الـ "API"

```
https://raw.githubusercontent.com/USERNAME/REPO/main/news.json
```

غيّر `USERNAME` و`REPO` باسمك واسم المستودع. هذا الرابط يرجع JSON مباشرة،
ويتحدث تلقائيًا كل ساعة بدون أي تدخل منك.

## شكل البيانات

```json
{
  "updated_at": "2026-06-16T15:00:00+00:00",
  "total_articles": 120,
  "sources": [
    {"source": "الجزيرة", "status": "ok", "count": 25, "feed_url": "..."}
  ],
  "articles": [
    {
      "source": "الجزيرة",
      "title": "...",
      "link": "...",
      "published": "...",
      "summary": "..."
    }
  ]
}
```

## التجربة على جهازك قبل الرفع (اختياري)

```bash
pip install -r requirements.txt
python fetch_news.py
```

سينتج لك `news.json` محليًا، تقدر تتأكد منه قبل رفع المشروع.

## تخصيص المصادر

عدّل القاموس `SOURCES` في أول `fetch_news.py`. أضف اسم المصدر ورابطه (رابط
RSS مباشر أفضل، وإلا ضع رابط الموقع الرئيسي وسيحاول السكربت يكتشف الرابط).

## تغيير وتيرة التحديث

عدّل سطر `cron` في `.github/workflows/update-news.yml`. مثلًا `*/30 * * * *`
لتحديث كل 30 دقيقة (انتبه: GitHub Actions على الحسابات المجانية له حد أقصى
لدقائق التشغيل بالشهر، وكل ساعة وتيرة معقولة جدًا وبعيدة عن أي حد).
